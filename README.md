## Fonctionnalités

> [!warning]
> Si les méthodes ci-dessous ne vous conviennent pas, n'hésitez pas à en ajouter de nouvelles. Toutefois, pour les débuts du projet, mieux vaudrait rester sur des bases simples.

Afin de gagner du temps et de réviser un peu plus efficacement, ce dépôt utilisera [obsidian](https://obsidian.md/) pour mettre en forme les notes, tandis que le partage se fera via [Github](https://github.com/).

Parmi les méthodes possibles, voici celles qui sont / seront supportées dans l'avenir :

| Logiciel                          | Type                             | Support | Branche |
| --------------------------------- | -------------------------------- | ------- | ------- |
| [Obsidian](https://git-scm.com/)  | Prise de note                    | Complet | Main    |
| [Anki](https://apps.ankiweb.net/) | Cartes de révision intelligentes | Complet | /       |

## Contributions

> [!TIP] 
> Avant toutes contributions sur la branche principale, n'hésitez pas à créer votre propre branche et de faire une pull request plus tard.

Sachant que l'objectif du projet est de pouvoir réviser en vue des exams, il est important qu'on fasse un petit peu attention à ce qu'on y ajoute.

Ce pourquoi, voici quelques règles afin de s'assurer que tout se passe bien :
- Ajouter des notes propres uniquement.
- Séparer les concepts en notes différentes.
- Signaler l'utilisation de LLMs.
- Sourcer toutes infos externes aux polys / fichiers de cours.
- Faire vérifier/relire les infos.
- Utiliser les fonctions d'obsidian (liens entre notes, tags..)
- Ajouter des fichiers pour d'autres logiciels dans les branches adéquates.

*(Cette liste changera probablement, tant pour faciliter le procéder que pour utiliser d'autres fonctions de plugins comme les kanban.)*
